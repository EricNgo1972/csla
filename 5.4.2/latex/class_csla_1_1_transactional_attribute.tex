\doxysubsection{Detailed Description}
Marks a Data\+Portal\+\_\+\+XYZ method to run within the specified transactional context. 

Each business object method may be marked with this attribute to indicate which type of transactional technology should be used by the server-\/side \mbox{\hyperlink{class_csla_1_1_data_portal}{Data\+Portal}}. The possible options are listed in the \mbox{\hyperlink{namespace_csla_a7a94b01a8b9a756c9e5316ca03a6137a}{Transactional\+Types}} enum. 

If the Transactional attribute is not applied to a Data\+Portal\+\_\+\+XYZ method then the \mbox{\hyperlink{namespace_csla_a7a94b01a8b9a756c9e5316ca03a6137aae1ba155a9f2e8c3be94020eef32a0301}{Manual}} option is assumed. 

If the Transactional attribute is applied with no explicit choice for transaction\+Type then the Transaction\+Scope option is assumed. 

Both the Enterprise\+Services and Transaction\+Scope options provide 2-\/phase distributed transactional support.
