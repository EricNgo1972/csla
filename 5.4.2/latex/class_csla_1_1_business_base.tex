\doxysubsection{Detailed Description}
This is the base class from which most business objects will be derived. 

This class is the core of the CSLA .NET framework. To create a business object, inherit from this class. 

Please refer to \textquotesingle{}Expert C\# 2008 Business Objects\textquotesingle{} for full details on the use of this base class to create business objects.

Saves the object to the database. 

Calling this method starts the save operation, causing the object to be inserted, updated or deleted within the database based on the object\textquotesingle{}s current state. 

If \mbox{\hyperlink{class_csla_1_1_core_1_1_business_base_a342756e9cf08028186b6cecde15b4f31}{Core.\+Business\+Base.\+Is\+Deleted}} is true the object will be deleted. Otherwise, if \mbox{\hyperlink{class_csla_1_1_core_1_1_business_base_a84273283b7db180b67a105c0e83427df}{Core.\+Business\+Base.\+Is\+New}} is true the object will be inserted. Otherwise the object\textquotesingle{}s data will be updated in the database. 

All this is contingent on \mbox{\hyperlink{class_csla_1_1_core_1_1_business_base_ad67992c09ceb801e17e64350c289a55b}{Core.\+Business\+Base.\+Is\+Dirty}}. If this value is false, no data operation occurs. It is also contingent on \mbox{\hyperlink{class_csla_1_1_core_1_1_business_base_a1e368e12e88afd948b898b2c6e26dac3}{Core.\+Business\+Base.\+Is\+Valid}}. If this value is false an exception will be thrown to indicate that the UI attempted to save an invalid object. 

It is important to note that this method returns a new version of the business object that contains any data updated during the save operation. You MUST update all object references to use this new version of the business object in order to have access to the correct object data. 

You can override this method to add your own custom behaviors to the save operation. For instance, you may add some security checks to make sure the user can save the object. If all security checks pass, you would then invoke the base Save method via {\ttfamily base.\+Save()}.
